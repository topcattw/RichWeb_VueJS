## 為什麼要做

目前遊戲重新整理頁面後就會失去進度，玩家也無法把某一局帶到下一次再繼續。加入 localStorage 自動暫存與 JSON 檔案匯出匯入後，可以同時解決日常續玩與手動備份分享兩種需求，且不需要引入後端。

## 變更內容

- 新增自動暫存機制，在每次有效互動後將目前遊戲狀態寫入 localStorage。
- 新增手動匯出功能，可將目前遊戲狀態下載成 JSON 檔案。
- 新增手動匯入功能，可從 JSON 檔案載回遊戲狀態。
- 在匯入前顯示覆蓋提醒，說明會同時覆蓋目前畫面中的進度與 localStorage 暫存，並徵求使用者同意。
- 新增存檔資料驗證與版本欄位，避免損毀或不相容的 JSON 直接覆蓋遊戲狀態。

## 能力範圍

### 新增能力
- `json-save-load`: 管理 localStorage 自動暫存、JSON 匯出匯入、存檔驗證與覆蓋確認流程。

### 調整既有能力

無。

## 影響範圍

- 影響 [app.js](app.js) 的遊戲狀態序列化、還原、互動流程與錯誤處理。
- 影響 [index.html](index.html) 的遊戲管理操作區與匯入確認流程。
- 影響 [styles.css](styles.css) 的存檔操作提示與狀態呈現。
- 影響 [README.md](README.md) 的使用說明與存檔規則文件。
- 不新增後端依賴，僅使用瀏覽器原生 localStorage 與檔案下載/上傳能力。
