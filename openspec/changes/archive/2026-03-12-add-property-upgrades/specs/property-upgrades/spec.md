## ADDED Requirements

### Requirement: 地產支援升級等級
系統應為每一個地產格保存升級進度資料，包含目前等級、最高等級、升級成本資料，以及各等級對應的租金數值。

#### Scenario: 初始化地產升級資料
- **WHEN** 新的一局遊戲開始
- **THEN** 每一個地產格都應初始化為等級 0 的持有狀態，並具備明確定義的最高等級與可對應各升級等級的租金數值

### Requirement: 玩家可以升級自己持有的地產
若目前行動中的玩家持有某筆地產，且該地產尚未達到最高等級、玩家資金也足夠，系統應允許玩家在自己的回合中升級該地產。

#### Scenario: 升級已持有的地產
- **WHEN** 目前行動中的玩家選擇一筆符合升級條件、且自己資金足以支付的持有地產並確認升級
- **THEN** 系統應扣除升級費用、將該地產等級提升一級、更新該地產的實際租金，並在遊戲紀錄中寫入這次升級事件

### Requirement: 必須限制滿級與資金不足的升級行為
當地產已達最高等級，或玩家資金不足以支付升級費用時，系統應阻止該次升級操作。

#### Scenario: 拒絕無效的升級操作
- **WHEN** 目前行動中的玩家嘗試升級一筆已滿級，或升級費用高於自身可用資金的地產
- **THEN** 系統不得改變該地產狀態、不得扣除玩家金額，且應明確表示此升級目前不可執行

### Requirement: 租金應反映地產目前等級
當其他玩家停在某筆地產上時，系統應依據該地產目前的升級等級來收取租金。

#### Scenario: 收取升級後的租金
- **WHEN** 玩家停在另一位玩家持有、且等級高於 0 的地產上
- **THEN** 系統應依該地產目前等級對應的租金數值收費，而不是使用基礎租金
