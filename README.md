# 富旅之城 Monopoly Lite

這是一個使用 HTML、Vue 3 Global Build 與 Bootstrap 5 製作的單頁大富翁遊戲範例。專案不需要建置工具，直接開啟 `index.html` 就能執行。

## 快速開始

1. 開啟 `index.html`。
2. 選擇 2 到 4 位玩家並輸入名稱。
3. 按下「開始遊戲」。
4. 目前回合玩家按「擲骰子」進行移動。
5. 若停在無主地產，可選擇購買或略過。
6. 完成當前操作後按「結束回合」，交給下一位玩家。

## 玩法

- 支援 2 到 4 名玩家輪流操作。
- 每位玩家開局資金為 $1,200。
- 經過起點可獲得 $200。
- 落在無主地產可購買，落在他人地產需支付租金。
- 已持有地產可在自己的回合中升級，最高 3 級，升級後租金會提高。
- 機會格會從牌堆抽卡，卡牌可能帶來金錢變化或移動效果，牌堆抽空後會用棄牌堆重洗。
- 每次有效互動後都會自動暫存到瀏覽器 localStorage，並可手動匯出或匯入 JSON 存檔。
- 資金低於 0 即破產，名下地產會回到銀行。
- 最後一位未破產的玩家獲勝。

## 一回合流程

1. 目前回合玩家確認自己的現金、持有地產與可升級地產。
2. 按下「擲骰子」後，玩家會依總點數移動。
3. 若經過起點，會先領取起點薪資。
4. 抵達目的地後，系統會立即結算該格效果。
5. 若是無主地產，可選擇購買或略過。
6. 若是自己的地產，可在回合中決定是否升級名下地產。
7. 若沒有待處理操作，就可以按「結束回合」。

## 房屋升級規則

- 只有目前回合玩家可升級自己的地產。
- 每筆地產預設從 Lv 0 開始，最高可升到 Lv 3。
- 升級成本會依該地產的基礎升級成本與目前等級成長。
- 地產一旦升級，其他玩家踩到時會依目前等級支付對應租金。
- 這個版本不包含旅館、抵押、拍賣或同色壟斷條件。

## 卡牌系統規則

- 玩家踩到機會格時會從牌堆抽一張卡，並立即結算。
- 目前支援三類效果：金錢增減、相對移動、指定位置移動。
- 每張卡結算後都會進入棄牌堆。
- 當牌堆抽空時，棄牌堆會重新洗牌後回到牌堆。
- 因卡牌移動到另一個機會格時，不會連鎖再抽第二張卡。

## 介面說明

### 建立遊戲

- 可以設定玩家人數與每位玩家名稱。
- 若已有進行中的遊戲，再按一次開始遊戲會依目前表單設定重新開局。

### 目前回合

- 顯示目前輪到的玩家、現金、所在格子與最近一次骰點結果。
- 提供擲骰、購買地產、略過購買與結束回合等按鈕。
- 也會列出目前回合玩家名下所有可升級地產。

### 機會卡牌

- 顯示最近抽到的卡牌標題、說明與效果摘要。
- 同時顯示目前牌堆剩餘數量與棄牌數量。

### 資產面板

- 依玩家目前狀態顯示現金、地產數量與可升級筆數。
- 若玩家破產，會在這裡標示狀態。

### 事件紀錄

- 依時間倒序顯示最新事件。
- 內容包含移動、買地、收租、抽牌、破產與存檔載入等結果。

## 存檔規則

- 每次有效互動完成後，系統都會將最新局面自動暫存到瀏覽器 localStorage。
- 玩家可以手動將目前局面匯出成 JSON 檔案下載保存。
- 玩家也可以從 JSON 檔案匯入局面，但載入前系統會提醒此操作會覆蓋目前畫面中的進度與 localStorage 自動暫存。
- 若匯入的 JSON 缺少必要欄位、格式錯誤或版本不相容，系統會拒絕載入。
- 結束本局時，瀏覽器中的自動暫存也會一併清除。

## 存檔操作步驟

### 使用自動暫存

1. 正常操作遊戲即可，系統會在每次有效互動後自動更新瀏覽器暫存。
2. 重新整理頁面後，可按「載入自動暫存」恢復最近一次保存的局面。
3. 若不需要保留，可按「清除暫存」刪除瀏覽器中的自動暫存。

### 匯出 JSON 存檔

1. 在遊戲進行中按「匯出 JSON」。
2. 瀏覽器會下載一個包含目前完整局面的 JSON 檔。
3. 這份檔案可自行備份，之後可再匯入回遊戲中。

### 匯入 JSON 存檔

1. 按「匯入 JSON」並選擇一個存檔檔案。
2. 系統會先檢查檔案格式與版本是否有效。
3. 驗證通過後，畫面會先顯示存檔摘要與覆蓋提醒。
4. 按下「確認覆蓋並載入」後，才會正式覆蓋目前畫面進度與瀏覽器自動暫存。

## 執行方式

1. 直接用瀏覽器開啟 `index.html`。
2. 或使用任何靜態檔案伺服器在專案根目錄提供頁面。

## 常見問題

### 重新整理頁面後，遊戲進度還在嗎？

只要瀏覽器的 localStorage 沒被清除，且你沒有按「清除暫存」或「結束本局」，就可以用「載入自動暫存」恢復最近一次保存的局面。

### 匯入 JSON 存檔後，會發生什麼事？

匯入成功後，系統會同時更新目前畫面中的遊戲狀態與瀏覽器中的自動暫存，所以重新整理後會維持剛剛載入的那份局面。

### 為什麼我的 JSON 檔不能載入？

常見原因包括：檔案不是由本遊戲匯出、內容缺少必要欄位、版本不相容，或檔案本身已損毀。這些情況下系統會拒絕載入，避免破壞目前局面。

### 結束本局和重新開局有什麼差別？

- 結束本局：清空目前對局，並一併清除瀏覽器中的自動暫存。
- 重新開局：用目前表單設定直接開始一局新遊戲，並以新局面覆蓋舊的自動暫存。

## 檔案結構

- `index.html`: 頁面結構與 CDN 引入
- `styles.css`: 視覺樣式與棋盤版面
- `app.js`: Vue 狀態、規則與互動邏輯
- `openspec/specs/`: 目前生效中的能力規格
- `openspec/changes/archive/`: 已完成並封存的 OpenSpec 提案、設計、規格與任務清單

## OpenSpec 能力規格

- `game-setup-and-session`: 管理本機開局、玩家初始化、回合追蹤與勝負判定
- `board-turn-flow`: 管理擲骰移動、起點獎勵、格子效果與回合推進
- `property-economy`: 管理地產購買、租金結算、資金變化與破產淘汰
- `property-upgrades`: 管理地產升級等級、升級成本與分級租金
- `chance-card-deck`: 管理機會卡牌堆、抽牌、棄牌、重洗與卡牌效果
- `json-save-load`: 管理 localStorage 自動暫存、JSON 匯出匯入、存檔驗證與覆蓋確認
